# CLAUDE.md Stack Templates

Reference file for the `start` skill. When generating CLAUDE.md for a new project, read the section matching `DETECTED_STACK` and use it as the body — filling in project-specific values where shown in `[brackets]`.

## Available Stacks

- **Laravel** (PHP + TALL stack)
- **Next.js** (TypeScript + React)
- **Node.js** (Express, Fastify, Hono, etc.)
- **Python** (Django, Flask, FastAPI, etc.)
- **Go** (Gin, Echo, Fiber, Chi, net/http, etc.)
- **Expo** (React Native)
- **Generic** (undetected or custom stacks)

For Laravel projects, use `~/.claude/templates/CLAUDE.md` directly (fast path). For all other stacks, generate from the appropriate section below.

---

## Laravel

> Fast path: copy `~/.claude/templates/CLAUDE.md` and replace `[Project Name]` / `[project-folder]` placeholders.
> The template below mirrors that file for reference only.

```markdown
# [Project Name] — Claude Code Instructions

## Tech Stack

- **Framework**: Laravel (latest), PHP via Herd Pro
- **Frontend**: Tailwind CSS + Alpine.js + Livewire (TALL stack)
- **Database**: MySQL via Herd Pro
- **Testing**: Pest PHP
- **Assets**: Vite

## Development

### Local URL

`[project-folder].test` — served by Laravel Herd (HTTPS via `.test` domain)

### Common Commands

```bash
npm run dev                    # Start Vite dev server
herd php artisan migrate       # Run migrations
./sync-db.sh                   # Sync production DB to local
herd php artisan optimize:clear # Clear all caches
```

## Workflow

```
/plan → /plan-review → [clear context] → /plan-approved → /plan-check → /pr → /review → /address-pr → /finish
```

**Branch flow**: `feat/feature-name → homolog → main`

- Feature branches always target `homolog` (staging)
- `homolog → main` happens via PR after QA
- Direct pushes to `main` are blocked by hooks

## Testing Rules

**NEVER run the full test suite** — it takes too long and blocks other work.

Always use filtered or targeted tests:

```bash
vendor/bin/pest --filter="TestName"
vendor/bin/pest tests/Feature/Path/ToTest.php
```

Full suite → ask the user to run `ptp` in their terminal.

## Commits

Format: `<emoji> <type>: <description>` (present tense, lowercase)

```
✨ feat: add payment processing
🐛 fix: resolve duplicate email validation
🧪 test: add coverage for order service
🧹 chore: update dependencies
```

**Never** add AI signatures (`Co-Authored-By`, `Generated by`, etc.)
